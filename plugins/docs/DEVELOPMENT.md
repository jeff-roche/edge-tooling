# Plugin Development Guide

Guide to creating Claude Code plugins for the Edge Tooling marketplace.

## Plugin Structure

Every plugin follows the Claude Code plugin format:

```text
my-plugin/
├── .claude-plugin/
│   └── plugin.json       # Required: plugin manifest
├── skills/               # Optional: skill definitions
│   └── my-plugin/
│       └── SKILL.md
├── hooks/                # Optional: event hooks
│   └── hooks.json
├── agents/               # Optional: agent definitions
│   └── my-plugin.md
├── .mcp.json             # Optional: MCP server config
└── README.md             # Required: documentation
```

## Creating a Plugin

### Using the Generator

```bash
# Interactive (prompts for components)
./marketplace new my-plugin

# With flags
./marketplace new my-plugin --skill --hook --agent
```

Component options:

- `--skill` — SKILL.md for slash commands or auto-invoked capabilities
- `--hook` — hooks.json for event-driven automation
- `--mcp` — .mcp.json for external tool integration
- `--agent` — Agent definition for subagent spawning

### Manual Creation

1. Create the directory:

   ```bash
   mkdir -p plugins/my-plugin/.claude-plugin
   ```

2. Create `plugin.json`:

   ```json
   {
     "name": "my-plugin",
     "description": "What this plugin does",
     "version": "1.0.0",
     "author": { "name": "your-github-handle" },
     "homepage": "https://github.com/openshift-eng/edge-tooling",
     "license": "Apache-2.0"
   }
   ```

3. Add components (skills, hooks, agents, MCP config).

4. Add a `README.md`.

## Plugin Manifest (plugin.json)

### Required Fields

| Field | Description | Example |
|-------|-------------|---------|
| `name` | Unique ID (lowercase, hyphens) | `"ovn-topology"` |
| `description` | Short description | `"Visualize OVN topology"` |
| `version` | Semantic version | `"1.0.0"` |
| `author.name` | Maintainer | `"jeff-roche"` |

### Optional Fields

| Field | Description |
|-------|-------------|
| `homepage` | URL to documentation |
| `license` | SPDX license identifier |
| `repository` | Source repository URL |

## Components

### Skills (SKILL.md)

Skills extend Claude's capabilities. Place them at `skills/<name>/SKILL.md`.

```yaml
---
name: my-skill
description: When Claude should use this skill
user-invocable: true
---

# my-skill

Instructions for Claude...
```

Key frontmatter fields:

- `name` — Skill identifier (becomes `/my-skill`)
- `description` — Claude uses this to decide when to auto-invoke
- `user-invocable` — Whether users can invoke via `/my-skill`

#### Skill Template Variables

Within skill instructions, you can reference these runtime variables:

| Variable     | Description                                          |
|--------------|------------------------------------------------------|
| `$PLUGIN_DIR` | Absolute path to the installed plugin directory     |
| `$ARGUMENTS` | Arguments passed by the user after the slash command |

Example usage in a skill body:

```bash
!`bash "${PLUGIN_DIR}/command.sh" --name "${ARGUMENTS}"`
```

### Hooks (hooks.json)

Event-driven automation. Place at `hooks/hooks.json`.

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "echo 'hook ran'",
            "timeout": 30
          }
        ]
      }
    ]
  }
}
```

### MCP Servers (.mcp.json)

External tool integration. Place at `.mcp.json` in plugin root.

```json
{
  "mcpServers": {
    "my-server": {
      "command": "my-mcp-server",
      "args": ["--port", "3000"]
    }
  }
}
```

### Agents

Agent definitions for subagent spawning. Place at `agents/<name>.md`.

## Validation

```bash
./marketplace validate my-plugin
```

Checks:

- `.claude-plugin/plugin.json` exists and is valid JSON
- Required fields present (name, description, version, author.name)
- Version follows semver
- Plugin name matches directory name
- README.md exists
- At least one component present

## Publishing

> **Note:** `.claude-plugin/marketplace.json` is auto-generated by `catalog-update`. Do not hand-edit it.

1. Run `./marketplace catalog-update` to rebuild the marketplace catalog
2. Run `bash plugins/tests/marketplace_smoke_test.sh`
3. Commit with: `feat(plugins): add my-plugin`
4. Create pull request

## User Installation

Users install plugins via Claude Code's built-in plugin system:

```text
/plugin marketplace add openshift-eng/edge-tooling
/plugin install my-plugin
```

## Categories

- **cluster-ops**: Cluster lifecycle, health checks, upgrades
- **debug**: Troubleshooting and analysis
- **deploy**: Deployment automation
- **network**: Network diagnostics
- **operator**: Operator development
- **ci-cd**: CI/CD automation
- **util**: General utilities

## Best Practices

- Keep plugins focused (single responsibility)
- Write clear SKILL.md descriptions so Claude knows when to invoke
- Never hardcode credentials — use environment variables
- Validate prerequisites before executing
- Provide actionable error messages
- Test on oldest supported OpenShift version
